### Root Cause of Sync Issues (important for future reference)
- **Wrong approach**: allocating clip duration by character count → two layers of guessing stacked
- **Correct approach**: use `ffprobe` silence detection (`silencedetect=noise=-35dB:duration=0.3`) on actual VO audio
  - Paragraph boundaries = silences > 1.0s between para_end and next para_start
  - Stanza splits = largest silence WITHIN each paragraph
  - **Use `silence_end` as stanza split — NOT midpoint** — midpoint causes text to appear ~0.77s early
- Hardcoded `PARA_AUDIO_TIMINGS` in script with actual measured values:
  ```
  (0.00,  10.14, 12.96)  # Para 0
  (14.53, 15.65, 18.29)  # Para 1
  (19.79, 21.08, 24.87)  # Para 2
  (26.40, 30.90, 33.94)  # Para 3
  (35.02, 42.09, 43.65)  # Para 4
  ```
- Erase transition duration = actual gap between para_end and next para_start (not fixed 0.5s)
- WRITE_FRAC = 0.65 per stanza window — text completes at 65%, holds for 35%

## MWS Whiteboard v18 — APPROVED 2026-03-24

### What finally worked
- **Per-paragraph TTS audio** — each paragraph has its own audio file. Video duration = audio duration. No timing guesses.
- **Silence detection (first gap, not largest)** — `min_silence=0.3s`, `min_t = total * 0.10`, first gap after 10% mark = stanza split. Para 3 correctly splits at 1.55s ("So that's what we built.")
- **Short stanza 0 (≤30 chars / ≤5 words) appears instantly** — no animation. "What we built:" snaps on.
- **Stanza 1 writes in first 50% of its window** — all bullets on screen before VO finishes listing them.
- **raw_progress (0→1 over full clip)** used for all stanza timing — not write_progress (which is over 80% of clip).
- Script: `scripts/produce_mws_whiteboard.py` — committed as v2, git `4c79601`

### Key lessons
- Never use largest silence gap — use FIRST significant gap after sentence start
- Never allocate clip duration by character count — use actual TTS audio per paragraph
- Short stanzas should not animate — they should appear instantly and hold
- Stanza 1 write window should be 50% of remaining clip, not 100% (too slow)
