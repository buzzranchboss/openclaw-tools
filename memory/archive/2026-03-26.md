## WiFi Tune-Up — Technical Background (for future reference)
- Root cause: ISP routers ship with 2.4GHz mode set to b/g/n — the N component causes timing/compatibility conflicts with older IoT devices
- Fix: change 2.4GHz mode to b/g only in router admin panel
- Affected carriers confirmed: AT&T Fiber (BGW320/BGW210), Xfinity (XB8 — firmware update triggered it), Verizon FiOS (CR1000A — known firmware bug), Spectrum (band steering conflicts)
- Symptoms: 5GHz works fine; printers, cameras, smart plugs, Blink/Ring cameras, smart home devices drop or fail to connect; range seems shorter; devices connect but lose internet
- Dave's personal case: AT&T BGW router reset to defaults → lost 6 Blink cameras in woods → traced to b/g/n mode
- First paying client context: Kim Winstead (owner of S.Y. Wilson) — printer connectivity issue, fixed same day
